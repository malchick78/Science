Q
Are we to assume that all input will start with an Operator and not a number?
A
I think it will be useful to give more details on how to approach this assignment. Let <pre> denotes the prefix expression, then its definition can be written in a form of two rules:
  1. <pre> → i
  2. <pre> → <op> <pre> <pre>
where <op> is one of the four arithmetic binary operators: +, -, * and /, and "i" is an integer. From the two rules we see that a prefix expression can start with either an integer number or an operator. One can also see, that the definition is recursive. This is where the tree structure comes into play. A tree will be binary, because the operators are binary, if prefix expression starts with an integer, it will become a leaf in the tree, if it starts with an operator, it will become a subtree, a node holding the operator, and left and right subtrees that are themselves prefix expressions.
